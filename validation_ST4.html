Validation Page — Collaborative (ST4)

HTML validation evidence for Profile Page and Sitemap Page — completed collaboratively by Students 1, 2 & 3

🤝

Collaborative submission — Students 1, 2 & 3. This group has three members. The Profile page and Sitemap page were completed as a shared team effort. ST1 led layout and SVG planning. ST2 implemented JavaScript and SVG interactivity. ST3 handled testing, output sections, and ran validation.

User Profile Page — Validation Report

✅ No errors · Validated

profile.html was validated using the W3C Markup Validation Service and returned no errors. ST3 ran the validation and confirmed that the two-column CSS Grid layout, step buttons, progress bar, profile output sections, and confirmation banner all use valid HTML5 markup. The JavaScript prompt() interactions do not produce any HTML in the static source, so the validator correctly assesses only the base page structure. All ARIA attributes — including aria-label on step buttons, role="alert" on the confirmation banner, and aria-label on the profile output containers — passed validation without issue. The embedded <style> block raised one informational note, which is expected and does not constitute an error under the HTML5 specification.

Sitemap Page — Validation Report

✅ No errors · Validated

sitemap.html was validated using the W3C Markup Validation Service and returned no errors. ST3 ran the validation following the resolution of a merge conflict in the SVG legend by ST1 and ST2. The inline SVG element — including the viewBox attribute, all <a> node links, <rect>, <text>, <line>, and <marker> elements — validated as correct SVG 1.1 embedded within HTML5. The role="img" attribute, root <title> and <desc> elements, and all aria-label attributes on clickable nodes passed validation. The page banner, navigation, main landmark, and footer all validated correctly. One informational note was raised for the embedded CSS block, which is valid in HTML5 and does not constitute an error.
